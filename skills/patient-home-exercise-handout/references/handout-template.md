# 自主トレ説明用紙テンプレート

患者さん向けの説明用紙は、原則として次の順で構成する。
標準の作り方は、A4 用の HTML を先に作り、そのあと Gemini などで手動生成した画像を差し替えて仕上げる方法とする。

## 1. タイトル

- 例: `おうちで行うひざの運動`

## 2. 目的

- 何のために行う運動かを 1 から 2 文で書く。
- 例: `ひざを動かしやすくし、立ち座りを少し楽にするための運動です。`

## 3. 実施ルール

- 1 日の回数
- 痛みの目安
- 呼吸を止めないこと
- 無理をしないこと

## 4. 種目カード

各種目ごとに次を入れる。

- 種目名
- 画像、SVG 仮図、または図解
- 開始姿勢
- 動き方
- 回数 / 秒数 / セット数
- 注意点

画像運用の基本方針:

- まず患者向けの説明文とレイアウトを HTML で固める。
- 次に Gemini などで 1 種目ずつ画像を手動生成する。
- 生成した画像を HTML の各カードへ差し替える。
- 画像がまだない段階では、SVG 仮図でレイアウト確認をしてよい。

SVG 仮図を使うときの基本方針:

- まずは棒人間や単純な線画でよいので、姿勢と動きが明確に伝わる図を作る。
- 1 種目 1 図を基本にする。
- 動く方向は矢印で示す。
- 立位では、壁や机など支えの位置を図に入れる。
- 必要に応じて左右ラベルを入れる。

Gemini など外部画像生成を使うときの補足:

- Gemini の nanobanana や ChatGPT など外部画像生成を使ってよい場合は、患者向けのシンプルな説明図を優先する。
- 写実的すぎる画像より、姿勢と動きが一目でわかる図のほうを優先する。
- 1 枚の画像に複数動作を詰め込みすぎない。
- 左右差や動く方向が重要な場合は、矢印やラベルを追加する。

## 5. 中止・相談の目安

- 強い痛み
- 腫れの増加
- しびれや脱力の悪化
- 転びそうになるふらつき

## 6. 補足

- 担当療法士からの一言
- 次回確認したい点

## HTML 化するときの方針

- 1 種目 1 カードで視線移動を少なくする。
- 画像の横に文章を置くより、画像の下に短文を置くほうが読みやすい場合が多い。
- 印刷前提なら A4 幅で余白を広めに取る。
- 実画像を差し込む場合は、カード内で `object-fit: cover` を使って見出し重複を減らすトリミングを検討する。
- SVG 仮図は HTML に直接埋め込むと、印刷時に劣化しにくく調整もしやすい。
